Remote access to the running server:

example using the default configuration

Having not changed the default configuration of the server, the following applies:

If AnomicFTPD is running on a Macintosh or on a Unix system on IP 192.168.0.4 port 2121 and you want to access the server from a MS Windows system, enter the following line as URL in the MS Internet Explorer to access the non-public account 'bob':
ftp://bob:123password@192.168.0.4:2121/
If the server has enabled write access, you can use the file window in the Internet Explorer in the same way as the local (file-) Explorer: drag and drop files in/out the window to copy files to/from the remote (the Mac) file system.

If AnomicFTPD is running on a MS Windows system on IP 192.168.0.1 port 2121 and you want to access the windows machine from a Mac, start the Internet Explorer on the mac and enter the following URL to access the non-public account 'jim':
ftp://jim:456password@192.168.0.1:2121/
Unlike the Internet Explorer on Windows, the Macintosh Internet Explorer does not support copying files to the remote ftp server. You can use a shareware tool like RBrowser for Mac OS X to access the ftp server in write mode.

Anonymous accounts can also be accessed by omitting the account name and the password:
ftp://192.168.0.1:2121/
or
ftp://192.168.0.4:2121/
...and so on.
In that case the client accesses the 'anonymous' account and has (according to configuration) no write access.
