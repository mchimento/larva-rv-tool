Configuration:

Many different properties of the server and the user accounts can be defined in configuration files, which are:
ftpd.init- general parameters of the ftpd application
ftpd.groups - access right configuration for groups
ftpd.accounts- group and password file for users


file ftpd.init

welcome=<string>- the welcome message broadcasted to the clients
port=<integer>- the port number of the ftp daemon, usually 21 or 2121
clients=<ip-list>- comma-separated list of ip's or '*' for all
unixfilemask=<mask>- a unix mask for file access rights, like '666'
unixfoldermask=<mask>- a unix mask for folder access rights, like '777'
unixuser=<string>- the unix user account for newly created files
unixgroup=<string>- the unix group account for newly created files
loglevel=[0¦1¦2]- level of output of the daemon
caching=[true¦false]- switches file caching in the ftp server
createindex=[true¦false]- switches an on-the-fly 'index.html' file creation
currYearFormat=<format>- defines the date format for the dir command
prevYearFormat=<format>- the format is according to SimpleTextFormat
charcoding=<coding>- the default character coding

Example: 

welcome=WELCOME TO THE ANOMIC FTP SERVER
port=2121
clients=*
loglevel=2
unixfilemask=666
unixfoldermask=777
unixuser=admin
unixgroup=admin
caching=false
createindex=true
currYearFormat=MMM dd HH:mm
prevYearFormat=MMM dd  yyyy
charcoding=UTF-8


file ftpd.groups

<group_name>=<'r'|'-'><'w'|'-'><'x'|'-'>','<UU>':'<DD>','<root_path>

this defines read (r), write(x) or execution (x) rights under the condition of a specific upload (UU) / download (DD) ratio (ratio not yet implemented). The root path as given here appears then as '/' to the client

Example: 

usergroup=rw-,00:01,/
guestgroup=r--,00:01,/pub
admingroup=rwx,00:01,/
dosusergroup=rw-,00:01,C:\\
dosguestgroup=r--,00:01,C:\\pub
dosadmingroup=rwx,00:01,C:\\

On a DOS or WinNT (W2K/XP) System, the backslash '\' must occur double since this symbol acts as an escape character.


file ftpd.accounts

<user_name>=<group_name>','<password>

this assigns a specific user to a group and defines the password of that user.

Example: 

bob=usergroup,123password
jim=dosusergroup,456password
anonymous=dosguestgroup,*
admin=dosadmingroup,789password

If the password is a '*', then any word is accepted as password. Browsers use the word 'anonymous' as the user name to authentify anonymously at a ftp server (in all cases when the used omits an user name). It is therefore necessary and sound to define a user group with no-write access that is used as group for anonymous accounts and to specify that this account shall be accepted without any password (like in this example here).
