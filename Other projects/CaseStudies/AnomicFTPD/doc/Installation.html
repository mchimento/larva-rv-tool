Installation and Start-up of the server:

The single download file contains all software for the different supported platforms. For the different architectures and operation systems, the installation and start-up procedure is slightly different. In some cases it is necessary to download additional software for decompression or to provide the mandatory system requirements.

All persons installing this software should know about the TCP/IP environment of their machines and other machines in the network they are running in. The host-IP and the network mask on all machines must be configured in such a way that they can access each other in the network. If this is the first time that you establish an IP-connection between two computers, use the Problems and Solutions to get a guide through all possible problems.

After the installation is complete, it is highly reasonable to perform additional configurations as described in one of the next sections to define the ftp user account and access rights.


MS Windows 95/98/NT4/W2K/XP:

Installation:

  • make shure the JRE is installed. If not, download the latest version from http://java.sun.com/.
  • decompress the file AnomicFTPD_(version).tar.gz using WinZip from http://www.winzip.com or WinRAR from http://www.rarlab.com.

Start-up:

  • double-click the file 'ftpd.bat'.
  • If you have problems starting the server then you are probably using an old version of java (1.1.6 or below). Try to disable a just-in-time - compiler by using the wrapper 'ftpd_nojit.bat' instead.


any Unix system that supports JavaVM 1.1 and later:

Installation:

  • make shure the JRE is installed. If not, download the latest version from http://java.sun.com/.
  • decompress the file AnomicFTPD_(version).tar.gz using the commands 'gunzip AnomicFTPD_(version).tar.gz' and 'tar xf AnomicFTPD_(version).tar'.

Start-up:

  • in the folder 'AnomicFTPD_(version)' enter the command './ftpd.sh'.


G3/G4/G5 Macintosh with any Mac OS X:

you can either use the installation instructions as for "any Unix system that supports JavaVM 1.1 and later" (see below) or the more simple instructions here:

Installation:

  • decompress the file AnomicFTPD_(version).tar.gz using StuffitExpander

Start-up:

  • double-click the file 'ftpd.command'

hint:
  • you can also create an alias of the file 'ftpd.command' and place it on the desktop or in the startup folder


PowerPC Macintosh with Mac OS 8.1 or later:

Installation:

  • make shure the MRJ is installed. If not, download the latest version for PowerPC/Mac OS 8.1 and later from http://developer.apple.com/java/download.html.
  • decompress the file AnomicFTPD_(version).tar.gz using the tools MacGzip from http://mirror.apple.com/Mirrors/info-mac/cmp/ (latest version http://persephone.cps.unizar.es/general/gente/spd/gzip/) and sunTAR from http://mirror.apple.com/Mirrors/info-mac/cmp/.
  • decompress the file 'ftpd.os89app.sit' in the subdirectory 'Specials' using StuffitExpander from http://www.aladdinsys.com/.
  • move 'ftpd.os89app' from 'Specials' to 'Classes'. Move also the files ftpd.init, ftpd.accounts and ftpd.groups to 'Classes'
  • If you want to use also the remote-shutdown - feature for your Mac, decompress 'shutdown.os89script.sit' ind Specials and move 'shutdown.os89script' into the 'Classes' folder

Start-up:

  • in 'Classes', double-click the file 'ftpd.os89app'.

Psion:

These are the installation notes for Psion that hat been contributed by Chris S Handley:

  • You need some additional files from a package that Chris S Handley created. You find them as 'PsionRunAnomicFTPD' in the directory 'Specials'.
  • The supplied files just need to be put in the same folder as the Java class files. Tapping (opening) the RunAnomicFTPD.opo file will start it up in Console mode. The other file is the source code which allows users to customise the launcher (using Psion's built-in OPL language), and contains brief instructions on how to do so.
  • It would be possible to make AnomicFTP appear as a normal Psion app on the Extras bar, but this would require installing all of the files in a System subdirectory - not desirable for beginners. This difficulty could be avoided by using a standard Psion SIS installation file, but it would require someone with a Psion to compile the SIS each time you updated the Java files... And the SIS would be large since it would contain all the Java files. So in the end I conclude it is best to use the simple method I give in the previous paragraph.

Start-up:

  • Tapping (opening) the RunAnomicFTPD.opo file will start it up in Console mode.
